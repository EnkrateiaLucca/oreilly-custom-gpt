1. Custom GPTs — What & Why
4 Custom GPT Use Cases
  • Translators of Complexity — simplify domain knowledge for non-experts
  • Tools — automate repetitive tasks (extraction, formatting, scheduling)
  • Agents — autonomous task execution with minimal supervision
  • Collaborators — work alongside you as dedicated thought partners
When to Build a Custom GPT
  • Writing too many edits to system messages
  • Complex tasks need multiple specialized prompts
  • Need consistent behavior across sessions
  • Require specialized domain knowledge grounded
  • Tip: Search GPT store first; build only for unique needs

2. Claude Artifacts
5 Artifact Content Types
  • Documents — Markdown, Plain Text (reports, notes)
  • Code snippets — Python, JS, SQL, and more
  • Websites — single-page HTML apps (interactive, shareable)
  • SVG images & diagrams — charts, flowcharts, logos
  • Interactive React — buttons, forms, data visualizations
Artifact Features
  • Dedicated preview window alongside chat
  • Fully interactive and editable inline
  • Version history — iterate without losing prior versions
  • Export and sharing capabilities
  • Save to Claude Projects for organization
  • Collaborative editing
Regular Chat
  • Inline text in the conversation
  • Primarily text-based output
  • Static, non-editable
  • Best for shorter content
  • Limited visualization
  • No version history
Artifacts
  • Dedicated preview window
  • Multiple output formats
  • Fully editable inline
  • Best for substantial content (>15 lines)
  • Rich visualization (React, SVG, HTML)
  • Full version history

3. Claude Desktop + MCPs (Model Context Protocol)
What MCPs Add to Claude
  • Open standard by Anthropic for AI-to-tool connections
  • Access to local file system (read/write files)
  • Connect to databases — query and write data
  • Connect to external APIs (GitHub, Slack, calendars)
  • Control web browsers and apps
  • Enables fully personal AI assistants
MCP Setup (Claude Desktop)
  1. Install Claude Desktop app (claude.ai/download)
  2. Edit claude_desktop_config.json
  3. Add MCP server entries with command + args
  4. Restart Claude Desktop
  5. Verify tools appear in conversation UI

Key MCPs: filesystem · browser · GitHub · Slack · databases

4. Prompt Templates — 15 Reusable Templates
Template Variable Syntax
  • {VARIABLE_NAME} — simple placeholder; replace with your value
  • {VAR|opt1,opt2,opt3} — multi-option; choose one or use your own
  • {VAR|description} — single hint/example of what to provide

Example: {FORMAT|PDF,HTML,DOCX} → pick "PDF" or write "LaTeX"

How to Use a Template (4 Steps)
  1. Choose template from prompts/prompt-templates/
  2. Scan for all {VARIABLE} placeholders
  3. Replace each with your specific value
  4. Customize further — templates are starting points, not rigid rules

Prompt Engineering Best Practices
  • Structure: Role + Context + Task + Format — the 4 elements of a strong prompt
  • Few-shot: Include 1–3 input/output examples to guide the AI's style
  • Chain prompts: Break complex work into multi-step prompt sequences
  • Iterate: AI assistants are forgiving — start simple and refine based on output
  • Combine: Mix elements from multiple templates for hybrid use cases

6. Decision Framework — Which Tool for What?
Custom GPTs vs. Claude Artifacts
  • Custom GPT: Persistent specialized ChatGPT config; saves instructions + knowledge files for reuse across sessions
  • Artifact: Claude's dedicated output window for substantial, interactive, shareable content in a single session
  • Use Custom GPT when you need a deployable tool others can use
  • Use Artifacts when you need interactive output to iterate on in-session
Choose Your Path
  • Need consistent ChatGPT behavior + domain knowledge? → Custom GPT
  • Need interactive/shareable Claude output (app, chart, doc)? → Artifact
  • Need Claude to access files, apps, or APIs? → Claude Desktop + MCP
  • Need a reusable prompt structure anyone can fill in? → Prompt Template
  • Need long-document analysis (>100K tokens)? → Claude (200K–1M context)
  • Need image generation built-in? → ChatGPT (gpt-image-2)
Key Takeaways
  • Custom GPTs: Build specialized ChatGPT configurations with role, rules, knowledge files, and API actions — reusable by anyone
  • Claude Artifacts: Get interactive, editable outputs (HTML apps, React, SVG, code) with version history and sharing built in
  • Claude Desktop + MCPs: Connect Claude to your local files, databases, and APIs to build a personal AI assistant
  • Prompt Templates: Use {VARIABLE} syntax for reusable, customizable prompts across 5 domains (15 templates total)
  • Platform choice: Claude wins on context length + coding; ChatGPT wins on image gen + voice — both excel at core AI tasks
  • Iteration is key: AI assistants are forgiving — start simple, use templates as starting points, and refine based on output
